# Agent 调用的ES搜索接口

> **重要**: Agent分析出的需求资料（标题/描述/详情/标签/参数）**不对用户展示**，仅用于构建ES搜索条件。用户只看到AI的reply文字和商品卡片。

---

## Agent 完整输出JSON

```json
{
  "isJadeQuery": true,
  "reply": "根据您的需求，我为您匹配到以下货源：",
  
  "keyword": "冰种飘花翡翠手镯",
  "tags": "冰种,手镯,飘花,58圈口",
  
  "demandTitle": "冰种飘花翡翠手镯 58圈口",
  "demandDescription": "寻找冰种质地的飘花翡翠手镯，圈口58mm，预算10万元以内，要求无纹无裂",
  "demandDetail": "用户需求：天然A货翡翠手镯，种水要求冰种以上，飘花灵动，色泽清雅。圈口58mm，宽度14mm左右。预算10万元以内。要求无纹无裂，完美度高。适合日常佩戴。",
  "tagsList": ["冰种", "手镯", "飘花", "58圈口", "A货", "无纹无裂", "预算10万", "日常佩戴"],
  "specs": {
    "产品品类": "手镯",
    "核心卖点": "冰种飘花",
    "主石材质": "翡翠A货",
    "翡翠造型": "平安镯",
    "翡翠种水": "冰种",
    "翡翠颜色": "飘花",
    "配石材质": "",
    "镶嵌工艺": "",
    "瑕疵情况": "无纹无裂",
    "款式风格": "日常佩戴",
    "适用场景": "日常",
    "产品寓意": "",
    "镶嵌材质": "",
    "镶嵌配件": "",
    "尺寸规格": "58mm圈口"
  }
}
```

### 传给ES的字段（从上面提取）

| JSON字段 | 用途 | 对应ES参数 |
|----------|------|-----------|
| `keyword` | ES全文搜索 → title+description | `?keyword=冰种飘花翡翠手镯` |
| `tags` | ES标签过滤 | `?tags=冰种,手镯,飘花` |
| `reply` | 展示给用户 | 前端气泡 |

### 存库的字段（不对用户展示）

| JSON字段 | 存到 `jade_demands` 表 |
|----------|----------------------|
| `demandTitle` | demand_title |
| `demandDescription` | demand_description |
| `demandDetail` | demand_detail |
| `tagsList` | tags (JSON) |
| `specs` | specs (JSON) |

## 接口

```
GET /api/search?keyword={keyword}&tags={tags}&page=1&size=3
```

**公开接口，无需登录。**

---

## 入参说明

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | string | 是 | 搜索关键词，ES对title/description做全文匹配 |
| tags | string | 否 | 标签过滤，逗号分隔，精确匹配。例如 `冰种,手镯` |
| page | int | 否 | 页码，默认1 |
| size | int | 否 | 每页数量，Agent固定传3 |

---

## Agent 需要输出的JSON格式

Agent接收用户自然语言后，**必须严格输出如下JSON**（不要多余文字，不要markdown包裹）：

```json
{
  "isJadeQuery": true,
  "keyword": "冰种飘花翡翠手镯",
  "tags": "冰种,手镯,飘花",
  "reply": "根据您的需求，我为您匹配到以下货源："
}
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| isJadeQuery | boolean | `true`=翡翠需求继续搜索，`false`=非翡翠需求走固定回复 |
| keyword | string | **最重要**。从用户话里提取的核心搜索词，传给ES的keyword参数。建议提取：种水+颜色+品类+特征，例如 `冰种飘花手镯` |
| tags | string | 从用户话里提取的标签，逗号分隔。例如 `冰种,手镯,58圈口`。可以为空字符串 `""` |
| reply | string | 给用户看的回复语，1-2句话。搜索到结果时用"根据您的需求，我为您匹配到以下货源："，没结果时用"暂时没有匹配的货源..." |

## 非翡翠需求

当用户问题与翡翠**完全无关**时：

```json
{
  "isJadeQuery": false,
  "keyword": "",
  "tags": "",
  "reply": "我是高翠AI，专业找翡翠助手。您可以试着这样问：\n- 找5万左右的冰种翡翠\n- 我要A货，没预算上限"
}
```

**核心原则**：不要拒绝回答。即使问题表面与购物无关，也要发散思维联想翡翠购物场景，尽量走 `isJadeQuery: true`。

## 示例

| 用户输入 | Agent输出 keyword | tags |
|----------|-------------------|------|
| "10万预算帝王绿手镯" | `帝王绿手镯` | `帝王绿,手镯` |
| "冰种平安扣预算2万无纹无裂" | `冰种平安扣无纹无裂` | `冰种,平安扣` |
| "冰种翡翠吊坠送礼自用均可" | `冰种翡翠吊坠送礼` | `冰种,吊坠,送礼` |
| "你好" | `翡翠` | `""` （发散联想翡翠） |
| "今天天气怎么样" | `翡翠` | `""` （联想翡翠购物） |

## 后端处理流程

```
Agent输出JSON → 后端解析keyword和tags → GET /api/search → ES返回3张商品卡片 → 前端展示
```
