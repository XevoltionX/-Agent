# 高翠网 — 宝塔面板部署教程

## 服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2核 | 4核 |
| 内存 | 2GB | 4GB+ |
| 磁盘 | 20GB | 50GB+ |
| 系统 | CentOS 7+ / Ubuntu 20+ |

> 注意：2核2GB跑ES（Elasticsearch）很吃力，建议使用MySQL搜索模式。ES为可选组件，`mode: mysql` 时后端不会尝试连接ES，不会因缺少ES而启动报错。

---

## 一、宝塔面板环境安装

### 1.1 安装基础软件
宝塔面板 → 软件商店 → 安装：
- **Nginx** 1.22+
- **MySQL** 8.0
- **Redis** 7.0
- **JDK** 21（Tomcat→设置→JDK版本选21）
- **Elasticsearch** 8.x（可选，仅 `elasticsearch` 搜索模式需要）

### 1.2 安装 Node.js（前端构建）
```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt-get install -y nodejs
```
或宝塔 → 软件商店 → Node.js版本管理器 → 安装 20.x

### 2.1 创建数据库
宝塔 → 数据库 → 添加数据库：

## 三、后端部署

### 3.1 打包
```bash
cd stone-server
mvn clean package -DskipTests
# 生成 target/stone-server-1.0.0.jar
```

### 3.2 上传jar包
宝塔 → 文件 → `/www/wwwroot/gaocui/` → 上传jar包

### 3.3 配置 application.yml
在jar包同目录创建 `application.yml`（或修改jar内配置）：
```yaml
server:
  port: 8080

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/gaocui?useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: gaocui
    password: 你的数据库密码
    driver-class-name: com.mysql.cj.jdbc.Driver
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 20MB
  mail:
    host: smtp.163.com
    port: 465
    username: 你的邮箱
    password: 你的SMTP授权码
    properties:
      mail.smtp.auth: true
      mail.smtp.ssl.enable: true
  data:
    redis:
      host: localhost
      port: 6379

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true

gaocui:
  search:
    mode: mysql          # 低配服务器用mysql，高配用elasticsearch
  upload:
    path: ./uploads/
  llm:
    doubao-api-key: ${ARK_API_KEY}
    doubao-model: doubao-seed-2-0-mini-260428

alipay:
  app-id: 你的支付宝沙箱appId
  app-private-key: 你的应用私钥
  alipay-public-key: 你的支付宝公钥
  notify-url: http://你的natapp域名  # 内网穿透回调地址
```

### 3.4 启动
宝塔 → 网站 → Java项目 → 添加Java项目：
- 项目Jar路径：`/www/wwwroot/gaocui/stone-server-1.0.0.jar`
- 端口：8080
- JDK版本：21
- 启动命令：`java -jar stone-server-1.0.0.jar --spring.config.location=application.yml`

## 四、前端部署

### 4.1 构建
```bash
npm install
npm run build
# 生成 dist/ 目录
```

### 4.2 上传dist
将 `dist/` 目录上传到 `/www/wwwroot/gaocui/dist/`

### 4.3 配置Nginx
宝塔 → 网站 → 添加站点：
- 域名：你的域名（如 `xjx.nat100.top`）
- 根目录：`/www/wwwroot/gaocui/dist`

配置文件添加API反向代理：
```nginx
server {
    listen 80;
    server_name 你的域名;

    root /www/wwwroot/gaocui/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;        # SSE支持
        proxy_read_timeout 600s;    # AI回复可能很慢
    }
}
```

## 八、常见问题

### Q: 内存不足
2GB内存跑全套可能OOM。建议：
- **搜索模式切 `mysql`**：配置 `gaocui.search.mode=mysql`，ES 相关代码完全跳过，不占内存也不会因缺少 ES 而启动报错
- **后端JVM参数** `-Xmx300m`：限制堆内存

```bash
java -Xmx300m -jar stone-server-1.0.0.jar --spring.config.location=application.yml
```

### Q: 启动时报 Elasticsearch 连接错误
检查 `application.yml` 中 `gaocui.search.mode` 的值：
- 设为 `mysql` 即可，后端完全跳过 ES，无需安装 ES 也不会报错
- 如果确实要用 ES，确保 ES 服务已启动且 `elasticsearch.host/port` 配置正确

### Q: AI回复很慢
豆包mini模型响应时间约5-15秒，属正常范围。可尝试：
- 切更小的模型（如 `doubao-lite`）
- 给前端加更长的等待提示

### Q: 支付宝回调不到
需natapp内网穿透，或使用有公网IP的服务器。

### Q: 域名访问不了前端
检查宝塔→安全→防火墙，放行80端口。
