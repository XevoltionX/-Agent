## 一、技术栈决策

### 前端

| 决策项 | 选型 | 理由 |
|--------|------|------|
| 前端框架 | Vue 3 + Vite | 轻量、H5最佳体验、PRD明确仅移动端 |
| UI 策略 | Vant 4（骨架）+ 手写（视觉核心） | Vant负责NavBar/Tabs/Toast/Dialog/Uploader/Checkbox，核心视觉（聊天气泡、商品卡片、数据面板等）全部手写CSS |
| CSS 方案 | Scoped CSS | 简单直接，无需额外依赖 |
| 状态管理 | Pinia | Vue 3官方推荐，模块化，轻量 |
| 路由 | Vue Router 4 | 标准方案 |
| HTTP 客户端 | Axios | 拦截器统一处理token/错误 |
| 图标 | Vant Icon + 自备SVG | Vant图标覆盖基础场景，特殊图标用SVG |
| 构建 | Vite | 标配，快速HMR |

### 后端

| 决策项 | 选型 | 理由 |
|--------|------|------|
| 后端框架 | Spring Boot 3 + Maven | Java生态标准，Bro技术栈 |
| 数据库 | MySQL 8 | 用户/商品/客资/通知等关系型数据 |
| 搜索引擎 | Elasticsearch 8 | 翡翠商品全文搜索+标签匹配推荐 |
| 缓存 | Redis（可选） | 验证码过期、登录token缓存 |
| ORM | MyBatis-Plus | 国内Spring Boot生态主流，CRUD高效 |
| API 风格 | RESTful | 标准前后端分离 |
| 存储 | 本地文件系统 | 商品图片上传存储（后续可迁移OSS） |
| AI Agent | 预留接口，最后阶段接入 | 自然语言→ES搜索条件翻译 |

## 五、VIP / 免费 差异逻辑

所有差异通过 `useAuthStore().isVip` 控制，不拆分路由：

| 页面 | VIP | 免费 |
|------|-----|------|
| 商家后台 | VIP徽标 · 上限100 · 邮箱完整 | 无徽标 · 上限2 · 邮箱打码 |
| 个人中心 | 有效期至日期 | 有效期至永久 |
| 发布商品1 | 限额文案100 · 达限toast"请下架" | 限额文案2 · 达限toast"需升级VIP" |
| 商品管理 | 底部"+发布新商品" | 底部"升级VIP发布更多商品" |
| 客资列表 | 邮箱完整 · 无底部按钮 | 邮箱**** · "升级VIP"按钮 |
| 客资详情 | 邮箱可见 · 复制+标记已联系 | 星号打码 · "升级VIP"按钮 |
| 账户权限 | 金白渐变卡 · VIP权限表 · 续费 | 灰白渐变卡 · 免费权限表 · 升级 |

## 六、关键PRD约束

1. **一屏展示**：每一页所有内容必须能在移动端一屏内展示完
2. **Chat持久化**：聊天记录存localStorage，强刷新才清空
3. **图片路径修正**：PRD内所有 `D:\桌面文件\...` 路径以 `docs/预览图/` 实际文件为准
4. **缺失文档**：前后端方案、AI-PIPELINE、买家匹配AGENT、知识库均不存在，架构决策自行完成
5. **登录后默认角色**：Mock阶段，登录默认进免费商家后台
6. **图片与描述冲突**：以PRD文字描述为准
7. **首页双模式**：游客（未登录）vs 商家（已登录），"商家入驻"按钮跳转目标不同

## 十一、ES 索引设计

### products 索引

```json
{
  "mappings": {
    "properties": {
      "id":           { "type": "long" },
      "merchant_id":  { "type": "long" },
      "title":        { "type": "text", "analyzer": "ik_max_word", "search_analyzer": "ik_smart" },
      "description":  { "type": "text", "analyzer": "ik_max_word", "search_analyzer": "ik_smart" },
      "detail":       { "type": "text", "index": false },
      "tags":         { "type": "keyword" },
      "price":        { "type": "double" },
      "price_range":  { "type": "keyword" },
      "status":       { "type": "keyword" },
      "created_at":   { "type": "date" }
    }
  }
}
```

- **IK分词器**做中文全文搜索（标题、描述）
- **tags** 用 keyword 精确匹配 + 多字段聚合（帝王绿、冰种、手镯…）
- **price_range** 预处理预算区间（如"1-5万"、"5-10万"）
- **status** 过滤：仅 `PUBLISHED` 出现在搜索结果
- 数据同步：MySQL商品新增/编辑/上下架时同步写入/更新ES

```json

  "message": "success",
  "data": {}
}
```

错误码约定：
- 200 成功
- 401 未登录/token过期
- 403 权限不足（如免费商家超限发商品）
- 400 参数校验失败
- 500 服务器异常

## 十三、业务流程（后端视角）

### 13.1 登录注册
```
用户输入邮箱 → POST /send-code 发验证码 → 用户输入验证码 → POST /login 
→ 后端查merchants表，无则自动注册(免费)，有则登录 
→ 生成token(含id+email+isVip) → 返回前端
```

### 13.2 商品发布
```
商家上传图片 → 前端调用AI识别(暂Mock) → 编辑信息 → POST /products/publish
→ 后端校验发布余额(VIP≤100,免费≤2) → 写入MySQL → 同步ES → 返回成功
```

### 13.3 ES搜索（当前阶段无AI）
```
首页输入框 → GET /search (keyword+tags) → ES全文检索 → 返回商品列表
→ 前端包装为"为您找到以下优质货源"卡片
```
最终阶段接入AI后：
```
用户自然语言 → POST /ai/match → LLM提取(品类/预算/品相/尺寸) 
→ 构建ES查询 → 返回Top3卡片
```

### 13.4 买家留资
```
商品详情页提交 → POST /products/:id/inquiry(可归类到leads) 
→ 写入leads表 → 查关联merchant_id → 写入notifications(新客资通知)
```
